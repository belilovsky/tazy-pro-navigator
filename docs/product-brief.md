# TAZY.PRO Project Navigator

**Статус:** первый самостоятельный прототип нового проекта, не сайт про породу тазы.

## Что собираем

TAZY.PRO Project Navigator — закрываемый цифровой пакет для инвестора, банка/Damu, акимата, QazInnovations, проектировщика, поставщиков оборудования, сырьевого и коммерческого партнёра.

Это не рекламный лендинг. Первый экран должен сразу работать как инвестиционно-инженерный cockpit:

- режимы просмотра по аудиториям;
- производственная цепочка `Stage 1 / Stage 2 / Stage 3 / Full cycle`;
- cutaway-завод и модули `M0–M16`;
- инженерные точки `P1–P10`;
- матрица оборудования;
- сценарная финансовая демо-модель;
- рынки и продуктовая архитектура;
- HACCP / QR-паспорт партии;
- TAZY.CARE / ESG;
- гейты финансирования;
- document room.

## Экосистемное решение

Чтобы не изобретать велосипед:

- стек оставлен простым: чистый HTML/CSS/ESM без сборки;
- данные вынесены в `src/data.js`, UI-логика в `src/app.js`;
- стили построены на AV DS-подобной token boundary: `--color-*`, `--radius-*`, `--ring-color`;
- проект не смешивается с `tazy.dog`, который относится к породной платформе;
- текущие GPT-зарисовки используются как визуальные ассеты и референсы, но интерактивность живёт в коде.

## Что намеренно не делаем в первом проходе

- Не подключаем React/Vite/Plotly, пока статический прототип покрывает задачу инвесторского навигатора.
- Не делаем настоящую авторизацию и backend data room до решения о хостинге и контуре доступа.
- Не фиксируем финальные банковские числа: финансовый блок помечен как управленческая демо-модель.
- Не подменяем ПСД, КП поставщиков и техусловия визуальным интерфейсом.

## Ближайшие следующие шаги

1. Уточнить домен и приватный доступ: пароль на nginx/Cloudflare Access/qdev auth.
2. Подключить реальные документы из data room.
3. Отдельно собрать `financial-model.json` из утверждённой модели.
4. Заменить черновые визуальные ассеты на согласованные рендеры.
5. Добавить экспорт отчёта для инвестора/банка.
